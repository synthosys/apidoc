You need to log into Access NSF or Access Workspace in order to pull API queries.

The link is in the form of: http://nsf.api.smetrics.org/v1/[[object]]?[[params]]

To limit the parameters, you use “=” after the specific parameter followed by the search term.

Proposal information is retrieved using the GET Method.

Operators allowable in general request formula (*note: “year” [[params]] has extra operators, explained below):

Operator Description Role Example in [[params]]
= equal sign

define or limit a parameter

 id=269662605
& add link multiple parameters id=000158971&page=patent
, comma add terms within a parameter, acts as “or” t1=39,397,328

[[object]]

Name Description
user The individual object is known as the user. These are the grant submitters. 
organization The user's academic organization. Can also be written as org.
proposal The user's submitted NSF proposal.
panel A panel is a group of reviewers.
topic Topic model keywords. They are compiled based on collected data.
patent Patent data. (Up to date to 2010 only. Currently, no real patent data exists, please see Lee Fleming’s work: http://readidata.nitrd.gov/star/about_patents.php)

[[params]]

Name Description
id User or topic object's unique id. 
rid Reviewer id. 
pid Panel id.
year Associated year. Possible operations are >, <, - (“between”, i.e. 2001-2011), >=, <=, and =.Default uses award year. To specify request year, insert "R" before the parameter value.
page An individual page. May use objects (with “=”) such as org (organization), prop (proposals), pi (also known as co-pi’s, or collaborators), patent (patents), grant (grants), search (often used with “q”, see below) and alias (possible double matches based on [[object]] name) to find connections.
q A basic search option yields up to 25 results. Used with page=search. Note: it looks for the search term only and ignores the id. (See Harvard example below.)
summ=[options] (or summ)

A summary of the user's files. summ (by itself) assumes that it is possible to summarize by the fields provided. For example, /topics?year=2000-2008&t1=10&summ will not show t1 or year details. It is mostly likely summarized on proposal, the most granular level. 

summ can set by status, year, topic (i.e. t1), org (division) and/or pge (multiples separated by “,”).
status The type of proposal data. You can choose between award, decline, or propose. If not specified, the default shows all options together.
format To download data in CSV format, add “format=csv” to the end of the query. 
legend Every page has a legend (list of labels). legend (by itself) gives only the labels. When you set legend=[label], you get a list of all the different [label]s. 
count Used in conjunction with other [[params]]. (Be sure to use “&”.) Shows how many instances the [[object]] occurs confined by those [[params]].


[[params]] only valid with topic [[object]]
t, t1, t2, t3, t4 The topics that describe a proposal. They are proposal specific, i.e. t1 refers to the most common topic, t4 refers to the least common topic (since there is a max of 4 topics). t refers to any topic. To use more than one topic in a query, you use “t” and then the topic number without spaces after, i.e. for topics 1 and 3, it would be written as: t13
org NSF divisions or program element code (pge). List with division codes at: http://www.nsf.gov/staff/orglist.jsp and also listed in the NSF organization list below. 

Notes

  • In terms of granularity: prop > pi > org
  • Can search multiple parameters using "&" and multiple terms within the parameters using ","
  • If query exceeds 1000 observations, error will appear
  • If you choose CSV format, be aware that excel will drop any “0” in front of a number. This means for proposal ids that begin with “0,” they may need to be converted back to their original form before they can be used.
  • Also, CSV format separates multiple topic ids with pipes in the same column instead of dividing them into separate columns
  • By default, entries with blank values will not be displayed;  those fields disappear.

 

*Please note: some outputs in this documentation show fake data to protect privacy!
