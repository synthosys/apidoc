The general request formula is http://nsf.api.smetrics.org/v1/[[object]]?[[params]]. Only change the object and parameter values! The object you are searching for is “organization”. The search parameter that you use is “q” since you are asking for a simple search query looking for organizations with the name “Harvard”.  Since in order to use “q”, you need to use with “page=search”, the url becomes: http://nsf.api.smetrics.org/v1/organization?page=search&q=Harvard. (You use “&” to combine the two parameters together.)

Reading the output. you see a “count” of 25 (since the query is capped at 25). This list shows the unique “nsf_id” (National Science Foundation identifier) along with the “name” of the organization. Some organizations that appear include: "MUSEUM OF COMP ZOO HARVARD UNIV", "President and Fellows of Harvard College", "PRES AND FELLOWS OF HARVARD COLLEGE", "KENNEDY SCH OF GOV DAS HARVARD UNIV", and "Harvard University Center for Hellenic Studies (Washington DC)". There may be some other information listed about each organization, such as its address.

What happens if you add in an organization id?

If you decide to add in organization 0013706000 into the search query, you set the id equal to the id number:

Try It! http://nsf.api.smetrics.org/v1/organization?id=0013706000&page=search&q=harvard

This results in the same output as without an organization id (such as above). Therefore, the search term does not take into account the id of the organization.

What if you want to export the data into excel?

Try It! http://nsf.api.smetrics.org/v1/organization?id=0013706000&page=search&q=harvard&format=csv

The same link with “&format=csv” in the end allows you to download the data into CSV file, usable in excel. (Remember that nsf_id numbers sometimes get guessed improperly with this method as leading zeros are dropped. You will have to add them back in in order to use nsf_id numbers.)
